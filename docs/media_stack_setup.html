Prerequisites & Folder Structure

ℹ️
Single share is critical. All download clients, Radarr, Sonarr, and Jellyfin must map the same root path to enable hardlinks — preventing file duplication when media is imported.

Required Plugins

PluginWhere to Install
Community ApplicationsUnraid Apps tab (usually pre-installed)
Docker Compose ManagerOptional — useful for grouped restarts

Recommended Folder Structure — /mnt/user/data/

/mnt/user/data/
├── torrents/
│   ├── movies/
│   ├── tv/
│   └── incomplete/
├── usenet/
│   ├── movies/
│   ├── tv/
│   └── incomplete/
└── media/
    ├── movies/
    └── tv/

Radarr/Sonarr hardlink completed downloads from torrents/ or usenet/ into media/. This only works if all containers share the same root path.

PUID / PGID

Find your values by opening an Unraid terminal and running:

id nobody

Typically PUID=99 and PGID=100 on Unraid. Use these for all containers.

Create the Folder Structure

In the Unraid terminal:

mkdir -p /mnt/user/data/{torrents/{movies,tv,incomplete},usenet/{movies,tv,incomplete},media/{movies,tv}}
chown -R nobody:users /mnt/user/data

Gluetun VPN Gateway

Gluetun is a VPN client container. qBittorrent and SABnzbd will share its network stack, routing all traffic through the VPN tunnel.

⚠️
Start Gluetun first. It must be running before qBittorrent and SABnzbd start — those containers depend on its network.

Tips & Troubleshooting

🔴 Gluetun Not Connecting

  • Verify VPN credentials and that your provider name exactly matches Gluetun's provider list
  • Check container logs: Docker tab → Gluetun → Log icon
  • Confirm /dev/net/tun is mapped and Privileged is enabled
  • Some providers require a specific SERVER_COUNTRIES or SERVER_CITIES format

🔴 qBittorrent / SABnzbd Web UI Not Loading

  • Confirm Gluetun is running first — dependent containers will fail without it
  • Ports must be mapped on Gluetun, not on qBittorrent/SABnzbd
  • Restart Gluetun first, then the download clients

🔴 Hardlinks Not Working (Files Being Copied)

  • Verify qBittorrent, SABnzbd, Radarr, Sonarr all map /mnt/user/data → /data
  • All containers must see the same absolute path — /data/torrents and /data/media must be on the same filesystem
  • Verify: docker exec radarr ls /data should show torrents, usenet, and media

🔴 Radarr/Sonarr Can't Reach Download Clients

  • Use gluetun as the hostname — not the Unraid IP
  • Confirm port mappings on Gluetun are correct
  • Check that all containers are on the same Docker bridge network

🔴 Permissions Issues — Files Not Importing

  • Ensure all containers use identical PUID/PGID values
  • Run in terminal to reset ownership:
chown -R nobody:users /mnt/user/data

✅ Recommended Container Start Order

OrderContainerReason
1GluetunMust be running before download clients
2qBittorrent, SABnzbdDepend on Gluetun network

💡 Pro Tips

  • Port forwarding: If your VPN provider supports port forwarding (Mullvad, PIA), configure the forwarded port in Gluetun and use it as qBittorrent's listening port for significantly better peer connectivity
  • Quality profiles: Set up custom quality profiles in Radarr/Sonarr (e.g., 1080p Remux, 4K HDR) before adding movies/shows so everything downloads at the right quality from the start
  • Real-time monitoring in Jellyfin: Enable "Monitor for new files" in library settings so new imports appear instantly without waiting for a scheduled scan
  • Recyclarr: Consider adding the Recyclarr container to automatically sync TRaSH Guide quality profiles and custom formats to Radarr/Sonarr
  • Backup appdata: Use the CA Backup / Restore Appdata plugin to schedule regular backups of your /mnt/user/appdata/ folder
