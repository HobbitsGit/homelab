Unraid

Full Rebuild Guide

Complete instructions for rebuilding the Unraid media stack from scratch — hardware failure, USB corruption, or fresh start. All media containers use a custom Docker network (media_stack) with container-name resolution.

🏠

Overview

This server runs a self-hosted media automation stack on Unraid. All media containers communicate on a custom Docker network (media_stack) using container names as hostnames. Download clients route through Gluetun for VPN privacy. Quality profiles are managed by Profilarr via the Dictionarry database. Subtitles are handled by Bazarr. Media is served by Jellyfin.

Post-Install — Initial Password

Pi-hole v6 generates a password on first run. If you miss it or need to reset:

Pi-hole v6 uses session cookies for auth. If login fails in Firefox, clear cookies for that site (padlock icon → Clear cookies and site data) — old br0 session data can conflict.

Security — Limiting User Access via ACLs

In the Tailscale admin console → Access Controls, restrict which devices can reach which ports. Example — limit a family member to Jellyfin only:

{
  "acls": [
    {"action": "accept", "src": ["tag:admin"], "dst": ["*:*"]},
    {"action": "accept", "src": ["tag:family"], "dst": ["tag:server:8096"]}
  ],
  "tagOwners": {
    "tag:admin": ["autogroup:owner"],
    "tag:family": ["autogroup:owner"],
    "tag:server": ["autogroup:owner"]
  }
}
⚠️
If giving Tailscale access to other users, use ACLs to restrict them to only the ports they need. A compromised device could otherwise reach all services.

Fan Control Setup

The MSI PRO B650M-A WiFi uses a Nuvoton NCT6687D Super I/O chip. The Linux kernel incorrectly loads the nct6683 driver for this chip, which runs in read-only mode and blocks all PWM writes. The correct driver is provided by the Nuvoton NCT6687 Driver plugin by ich777.

Hardware Layout

ComponentConnectionPWM Channel
CPU FanCPU_FAN header (direct)pwm1 — BIOS controlled, do not touch
6× Case Fans via NA-FH1 hubSYS_FAN1 headerpwm3 — autofan controlled

Install Order

1

Install Nuvoton NCT6687 Driver from Community Applications (by ich777)

2

Reboot — driver loads automatically, nct6683 is blacklisted

3

Install Dynamix System Autofan from Community Applications

4

Configure Fan Control in Settings → Fan Control (see table below)

5

Install User Scripts and create the autofan_restart boot script (see below)

⚠️
The NCT6687 Driver plugin must be installed and the server rebooted before configuring Autofan. Installing in the wrong order will result in the wrong PWM paths being saved.

Autofan Configuration

Go to Settings → Fan Control and apply these settings:

SettingValue
Fan control functionEnabled
PWM controllernct6687 - pwm3
PWM fannct6687 - fan1_input
Minimum PWM valueClick Detect (Noctua fans detect as ~5)
Low temperature threshold35°C
High temperature threshold45°C
Refresh interval5 minutes
Exclude drivesNone
ℹ️
Autofan monitors the highest SMART temperature across all array drives and scales fan speed linearly between the low and high thresholds.

Config File Location

Autofan saves its config to flash at:

/boot/config/plugins/dynamix.system.autofan/dynamix.system.autofan2.cfg

Expected contents after configuration:

options="-c /sys/devices/platform/nct6687.2592/hwmon/hwmon4/pwm3 -f /sys/devices/platform/nct6687.2592/hwmon/hwmon4/fan1_input -l 5 -t 35 -T 45 -m 5"
service="1"
controller="/sys/devices/platform/nct6687.2592/hwmon/hwmon4/pwm3"
fan="/sys/devices/platform/nct6687.2592/hwmon/hwmon4/fan1_input"
pwm="5"
low="35"
high="45"
interval="5"
exclude=""

User Script — autofan_restart

Due to boot timing, autofan must be restarted after the nct6687 driver and array are fully initialized. Create this in Plugins → User Scripts:

SettingValue
Nameautofan_restart
ScheduleAt Startup of Array
#!/bin/bash
# Wait for system to fully initialize
sleep 60

# Ensure nct6687 driver is loaded
if ! lsmod | grep -q nct6687; then
    modprobe nct6687
    sleep 5
fi

# Ensure pwm3 is in manual mode
echo 1 > /sys/devices/platform/nct6687.2592/hwmon/hwmon4/pwm3_enable

# Restart autofan cleanly
/usr/local/emhttp/plugins/dynamix.system.autofan/scripts/rc.autofan restart

echo "autofan restarted at $(date)"

Unified /data Mount

All media containers (Radarr, Sonarr, qBit, SAB, Bazarr) use a single volume: /data → /mnt/user/data. This enables hardlinks and eliminates remote path mappings.

Troubleshooting

🔴 Pi-hole Login Fails in Firefox

Old session cookies from a previous network configuration (e.g. br0 IP) conflict with the new bridge setup.

  • Click the padlock icon in the address bar → Clear cookies and site data
  • Or try logging in via a private/incognito window first

🔴 Pi-hole Not Receiving Queries from IoT Devices

  • Confirm the IoT to PiHole firewall rule is above the Isolated Networks block rule in UniFi
  • Confirm IoT network DHCP DNS is set to 10.5.10.10 with Auto DNS Server unchecked
  • Reconnect IoT devices to force DHCP lease renewal
  • Check Pi-hole interface settings: Settings → DNS → Permit all origins
docker exec -it binhex-official-pihole pihole -t

🔴 Internet Stops Working When Pi-hole is DNS

Pi-hole is rejecting queries — usually the interface setting is too restrictive.

  • Go to Pi-hole GUI → Settings → DNS → Interface settings → Permit all origins
  • Check Pi-hole container is healthy: docker ps | grep pihole

🔴 Captive Portal / "No Internet" Warning on Phone

Pi-hole may be blocking Apple/Google connectivity check domains. Whitelist them:

docker exec -it binhex-official-pihole pihole allowlist captive.apple.com
docker exec -it binhex-official-pihole pihole allowlist www.apple.com
docker exec -it binhex-official-pihole pihole allowlist appleiphonecell.com
# Android:
docker exec -it binhex-official-pihole pihole allowlist connectivitycheck.gstatic.com

🔴 VPN Not Working

docker logs gluetun --tail 50
docker exec gluetun curl -s https://api.ipify.org
  • Verify ProtonVPN credentials
  • Try different SERVER_COUNTRIES
  • Check both Host/Container ports are filled on Gluetun

🔴 Container Can't Reach Another

  • Verify both on media_stack
  • Use container name, not IP
  • Download clients: use gluetun as hostname
  • Jellyfin: use actual Unraid IP (not on media_stack)

🔴 SABnzbd Rejecting Connections

Config → Special (Advanced Settings) → host_whitelist → add gluetun, radarr, sonarr

🔴 Can't Import Downloads

  • All containers must use /data → /mnt/user/data
  • Remove any remote path mappings
  • Check SAB completed = /data/usenet, qBit save = /data/torrents/

🔴 Docker Network Lost After Reboot

docker network create media_stack
# Settings → Docker → Advanced → Preserve user defined networks → Yes

🔴 Jellyfin Wrong Metadata

  • Folder names must include year: Breaking Bad (2008)
  • Series Folder Format: {Series Title} ({Series Year})
  • Anime library: Shows type with AniDB as top metadata provider

🔴 Container Becomes Orphan Image

Switching network types can break container config. Re-add from Apps tab with same appdata path, set to Custom: media_stack, re-add all port mappings. Data is safe.

🔴 Unplanned Overnight Reboots

Unplanned reboots trigger parity checks on 3× 28TB drives. Most common cause is plugin auto-updates. To prevent:

  • Go to Apps → Update Applications → Plugin Auto Update Settings
  • Set Auto Update All Plugins to No
  • Disable individual plugin autoupdate toggles in the list below
last -x | head -20
dmesg | grep -i "panic\|mce\|oom\|killed"
cat /var/log/apcupsd.events | tail -20
⚠️
The cron.daily job runs at 4:40 AM. A plugin with auto-update enabled can reboot the server silently at this time.

🔴 Fans Not Responding to Drive Temperature

  • Confirm correct driver: lsmod | grep nct — must show nct6687, not nct6683
  • If nct6683 is loaded, install the Nuvoton NCT6687 Driver plugin (ich777) from Community Applications and reboot
  • Confirm autofan is running: ps aux | grep autofan | grep -v grep
  • If autofan is not running, start manually: /usr/local/emhttp/plugins/dynamix.system.autofan/scripts/rc.autofan start
  • Confirm the autofan_restart User Script is set to At Startup of Array
  • Check pwm3 is not stuck at 0: cat /sys/devices/platform/nct6687.2592/hwmon/hwmon4/pwm3
  • Test fan response manually: echo 255 > /sys/devices/platform/nct6687.2592/hwmon/hwmon4/pwm3
