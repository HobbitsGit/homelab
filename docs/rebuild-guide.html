Unraid

Full Rebuild Guide

Complete instructions for rebuilding the Unraid media stack from scratch — hardware failure, USB corruption, or fresh start.

🏠

Overview

This server runs a self-hosted media automation stack on Unraid. Download clients route through a VPN container (Gluetun) for privacy. Media is automatically managed by Radarr/Sonarr and served by Jellyfin.

Phase 2 — Array & Storage Setup

2.1 Assign Drives

1

Go to Main tab

2

Set parity slots: 1, data slots: 2+

3

Assign: Parity → one 28TB, Disk 1 → 28TB, Disk 2 → 28TB

2.2 Add Cache Pool

1

Main tab → find the Pools section

2

Add a new pool and assign the 1TB NVMe

3

Pool will format as btrfs (default — leave as-is)

2.3 Start Array

1

Click Start on the Main tab

2

Format all drives when prompted — choose XFS for data drives

3

A parity build will begin — takes many hours on large drives, this is normal

ℹ️
The server is usable during the parity build, but performance will be reduced.

2.4 Configure 250GB SSD (Unassigned Device)

1

Install the Unassigned Devices plugin first (see Phase 8)

2

Main tab → scroll to Unassigned Devices → find the 250GB SSD

3

Enable Destructive Mode in Unassigned Devices settings (temporarily), format as XFS, then disable it

4

Click Mount, enable Auto Mount, name it Backup

5

Mount point becomes /mnt/disks/Backup

Phase 9 — Restore from Backup

If rebuilding from a backup (not from scratch), restore appdata before starting containers:

# Stop all containers first, then restore from backup archive
tar -xzf /mnt/disks/Backup/appdata/BACKUP_ARCHIVE.tar.gz -C /mnt/cache/appdata/
ℹ️
Start containers in order (see the Overview section). Verify each one before starting the next.

Troubleshooting

🔴 VPN Not Working

# Check Gluetun logs
docker logs Gluetun --tail 50

# Verify qBittorrent is using VPN
docker exec qbittorrent curl -s https://api.ipify.org
# Should return a ProtonVPN IP, not your home IP
  • Check ProtonVPN OpenVPN credentials are correct
  • Try a different SERVER_COUNTRIES value
  • Add DNS_ADDRESS=8.8.8.8 environment variable to Gluetun if DNS fails

🔴 Radarr/Sonarr Can't Import Downloads

1

Check remote path mappings: Radarr /downloads/movies → /data/usenet/movies, Sonarr /downloads/tv → /data/usenet/tv

2

Verify folders exist: 

ls /mnt/user/data/usenet/
# Should show: movies/ tv/ incomplete/

3

Check SABnzbd categories point to correct folders

🔴 Jellyfin Transcoding / Slow Playback

  • Install Jellyfin Media Player on PC for direct play (no transcoding needed)
  • For hardware transcoding, verify /dev/dri device is passed to the container
  • In Dashboard → Playback → Transcoding, set Hardware Acceleration to VAAPI
  • Seeking is slow on large remux files — this is normal behavior
  • Performance is impacted during parity checks — wait until complete

🔴 Container Won't Start — Duplicate Mount Point

Error: Duplicate mount point: /config

Edit the container and remove the duplicate path entry. The template may have auto-added a path that conflicts with one added manually.

🔴 SABnzbd Can't Be Accessed

SABnzbd defaults to port 8085 internally. If you can't access it:

# Check what port it's actually listening on
docker logs sabnzbd | grep "Starting web"
# Then access at http://GLUETUN_IP:PORT_FROM_LOGS

If the internal port doesn't match, edit /mnt/cache/appdata/sabnzbd/sabnzbd.ini and update the port = value, then restart the container.

🔴 Indexers Not Syncing to Radarr/Sonarr

  • In Prowlarr → Settings → Apps, verify Radarr and Sonarr connections test successfully
  • Click Sync App Indexers button
  • Check that indexer categories match what Radarr/Sonarr expect (movies vs TV)
