Updating Containers

Method 1 — Via Unraid Dashboard (Recommended)

1

Go to the Docker tab

2

Look for containers showing an update available badge

3

Click the container icon → Check for Updates

4

If an update is available, click Update

5

The container will pull the new image and restart automatically

Method 2 — Update All at Once

1

Go to Docker tab → click Check for Updates at the top

2

Review the list of available updates

3

Click Update All — or update individually to be safe

Best Practice

  • Update one container at a time when possible
  • After updating, verify the container starts and the web UI is accessible
  • Check logs after updating: click the container icon → Logs
  • For critical containers (Gluetun, SABnzbd, qBittorrent), verify VPN is still working:
docker exec qbittorrent curl -s https://api.ipify.org
docker exec sabnzbd curl -s https://api.ipify.org

Both should return a VPN IP, not your real home IP.

Updating Unraid OS

1

Go to Tools → Update OS

2

Review the release notes before updating

3

Unraid will download and stage the update

4

Schedule the update for a low-activity time

5

The server will reboot to apply the update

6

After reboot, verify all containers start correctly

7

Run a quick parity check if prompted

⚠️
Always back up your Unraid flash drive before a major OS version update.
🗄️

Backup Strategy 3-2-1

CopyLocationWhat
1Array (protected by parity)All media files
2250GB SSD (Unassigned Device)Appdata, flash config
3Backblaze B2 (offsite)Appdata, important documents

What IS backed up

  • Docker appdata (container configs, databases)
  • Unraid flash/USB config
  • Important documents share
  • VM configs (if applicable)

What is NOT backed up locally

  • Media files (movies, TV shows) — these can be re-downloaded
  • Photos — backed up to Backblaze directly

Appdata Backup Local SSD

The Appdata Backup plugin handles this automatically.

Schedule

SettingValue
FrequencyWeekly, Sunday at 3:00 AM
Destination/mnt/disks/Backup/appdata
Retention30 days

Manual Backup

1

Go to Settings → Appdata Backup

2

Click Start Backup

3

Monitor the log for completion

Verifying the Backup

  • Go to Main tab → scroll to Unassigned Devices
  • Check the Used column on the Backup SSD
  • Or browse to /mnt/disks/Backup/appdata via the Files tab to see archives

What Gets Backed Up

  • /mnt/cache/appdata/ — all Docker container config data
  • Unraid flash drive configuration
  • VM metadata

Flash Drive Backup

⚠️
The Unraid flash drive stores your array config, Docker templates, and settings. Losing it without a backup means reconfiguring from scratch.

Automatic Flash Backup (via Appdata Backup Plugin)

The Appdata Backup plugin backs up the flash drive automatically as part of its weekly run, to /mnt/disks/Backup/appdata.

Manual Flash Backup

1

Go to Main tab

2

Click on Flash in the Boot Device section

3

Click Flash Backup → Download

4

Save the ZIP file somewhere safe (external drive, cloud storage)

Unraid Connect Flash Backup

1

Go to Tools → Unraid Connect

2

Enable Flash Backup in the settings

3

Unraid will automatically back up your flash config to Lime Technology's servers

Restore Procedures

Restore Appdata from Local SSD

Use this when a container's config gets corrupted or you need to roll back.

1

Stop the affected container

Docker tab → click container → Stop

2

Rename the corrupted folder

mv /mnt/cache/appdata/CONTAINERNAME /mnt/cache/appdata/CONTAINERNAME.bak
3

Find the backup archive

ls /mnt/disks/Backup/appdata/
4

Extract the backup

tar -xzf /mnt/disks/Backup/appdata/BACKUP_FILE.tar.gz -C /mnt/cache/appdata/
5

Start the container and verify it works

Single Container Restore via Plugin

1

Go to Settings → Appdata Backup

2

Look for the restore option in the plugin UI

3

Select the container and backup date to restore from

Restore Appdata from Backblaze

Use this if the local SSD backup is also unavailable (catastrophic failure).

1

Install Duplicati or Rclone on a working system

2

Connect to your Backblaze B2 bucket

3

Download the relevant backup archive

4

Follow the same extraction steps as the local SSD restore above

Restore Unraid Flash / Config

If USB drive fails:

1

Get a new USB drive (minimum 1GB, quality brand)

2

Download the Unraid OS installer from unraid.net and flash to the new USB

3

Copy your flash backup ZIP contents to the new drive's /config folder

4

Boot from the new USB drive

5

License is tied to the USB GUID — transfer it at keys.unraid.net if needed

If config is corrupted (drive is fine):

  • Boot into Unraid → Tools → New Config (resets array assignments — use with caution)
  • Or restore specific config files from your flash backup ZIP

Parity Checks

Schedule

SettingValue
FrequencyMonthly, 1st of the month at 3:00 AM
TypeRead-only (non-correcting)
Configured inSettings → Scheduler → Parity Check

Manual Parity Check

1

Go to Main tab

2

Click Check next to the array

3

Choose Read-Only for routine checks; Correcting only if a previous check found errors

Interpreting Results

ResultMeaning
0 errorsEverything is healthy ✅
Errors found (first time)Run a correcting check to fix
Errors found repeatedlyPossible failing drive — check SMART data

Performance Impact

  • Array read/write speeds reduced by ~50% during check
  • Jellyfin streaming and transcoding may be affected
  • Scheduled for 3:00 AM to minimize impact
  • A 28TB parity check typically takes 8–16 hours

UPS Management

Hardware

ModelCyberPower CP1500PFCLCD
TypePure sine wave, 900W

Unraid Configuration

Go to Settings → UPS Settings to configure:

SettingValue
CableUSB
TypeCyberPower
Shutdown battery level25–30%

Power Outage Sequence

1

Power fails → UPS switches to battery

2

Unraid monitors battery percentage

3

When battery reaches configured threshold, Unraid initiates clean shutdown

4

All containers stop gracefully, array spins down safely

5

Power returns → manually power on the server

ℹ️
At ~130W load, the CP1500PFCLCD provides approximately 20–30 minutes of runtime — enough for a clean shutdown with buffer.

Quick Reference Commands

VPN Verification

# Verify VPN is working for download clients
docker exec qbittorrent curl -s https://api.ipify.org
docker exec sabnzbd curl -s https://api.ipify.org

Disk & Storage

# Check disk usage
df -h /mnt/user/data/

# List media folders
ls /mnt/user/data/media/movies/
ls /mnt/user/data/media/tv/

Container Management

# View container logs
docker logs CONTAINERNAME --tail 50

# Restart a container
docker restart CONTAINERNAME

# Check all running containers
docker ps

Drive Health

# Check NVMe health
nvme smart-log /dev/nvme0

# Check array drive SMART
smartctl -a /dev/sdX
